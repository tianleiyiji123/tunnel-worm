# 2026-03-29 工作日志

## 修复安装向导三个 Bug

### Bug 1: MySQL 测试连接 pymysql ImportError
- **问题**: Docker 容器内无 pymysql，点 MySQL 测试时报原始 `No module named 'pymysql'`
- **修复**: `routes/setup.py` test-db 端点将 `except Exception` 拆分为 `except ImportError` + `except Exception`，返回友好中文提示
- **结果**: `{"success": false, "message": "当前环境未安装 pymysql 驱动。Docker 镜像默认使用 SQLite，如需 MySQL 请自行安装 pymysql：pip install pymysql"}`

### Bug 2: 云存储测试 SDK ImportError
- **问题**: 选 MinIO/OSS/COS 测试时报通用 "缺少对应的存储 SDK"
- **修复**: `routes/setup.py` test-storage 端点 ImportError 捕获中解析具体缺失模块名，通过 sdk_map 返回具体的安装命令
- **结果**: `{"success": false, "message": "缺少存储 SDK：minio (pip install minio)。Docker 默认镜像未包含云存储 SDK..."}`

### Bug 3: 完成安装无响应
- **问题**: SQLite + local 组合点击"完成安装"无任何反馈
- **修复**:
  - 后端: `finish` 端点整体包裹 try-except，异常返回 `{"success": false, "message": "初始化失败: ..."}` 而非 500
  - 前端: `SetupView.vue` 三个 handler（handleTestDb/handleTestStorage/handleFinish）全部添加 `catch` 块，从 `e.response?.data?.detail` 提取错误信息显示
- **结果**: API 直接返回 `{"success": true, "message": "配置保存成功，系统初始化完成"}`

### Docker 验证
- `--no-cache` 重新构建镜像成功
- 清除容器内 config.json 后重新进入安装向导
- 三个端点全部通过 curl 验证

## 修复完成安装后不跳转首页

- **根因**: 路由守卫用 `_initialized` 缓存了初始化状态，页面首次加载时缓存为 `false`，`finishSetup` 成功后 2s 执行 `router.push('/')`，守卫读缓存仍是 `false`，把用户踢回 `/setup`
- **修复**: `router/index.ts` 导出 `markInitialized()` 函数设置 `_initialized = true`；`SetupView.vue` 在跳转前调用 `markInitialized()` 清除缓存
- 涉及文件: `client/src/router/index.ts`, `client/src/views/SetupView.vue`

## Docker 部署
- 构建镜像: `docker compose build --no-cache`
- 首次构建后需要手动迁移旧数据库（添加 transfer.user_id 和 transfer.permanent 列）
- 启动命令: `docker compose up -d`，端口 7895
- 健康检查: GET /api/health → {"status":"ok","service":"suisuichong","initialized":true}
- 认证接口: GET /api/auth/me → {"user":null}（未登录时）

## Bug 修复
- **弹窗被裁剪**: LoginDialog 从 AppHeader(fixed) 移到 App.vue 顶层，通过 provide/inject 全局控制
- **bcrypt 72 字节限制**: auth_service.py 新增 SHA-256 预哈希 _prehash_password()，hash/verify 都先预哈希
- **passlib + bcrypt 版本不兼容**: passlib 1.7.4 不兼容 bcrypt 5.x（缺少 __about__ 模块），在 requirements.txt 和 requirements-docker.txt 中固定 bcrypt==4.2.1
- **Logo SVG 比例**: viewBox 从 0 0 200 200 改为 22 62 137 82，虫子可见占比从 ~30% 提升到 ~90%
- Logo viewBox 统一: `22 62 137 82` (width=137, height=82, ratio ≈ 1.67:1)
- **Logo 全面统一**: HomeView、CodeDisplay、SetupView、RetrieveView、favicon 全部替换为橙红渐变虫子（与 AppHeader/LoginDialog 一致），移除了旧版绿色通用虫子图标

## 密码升级
- 密码从纯 4 位数字改为 4 位大写字母+数字混合（排除易混淆字符 O/0/I/1）
- `transfer_service.generate_code()`: 使用 `random.choices(chars, k=4)`，chars = A-Z(排除O/I) + 0-9(排除0/1)，共 32 个字符
- `PasswordInput.vue`: 移除 `inputmode="numeric"`，允许字母输入，自动转大写，CSS `uppercase`
- `CodeDisplay.vue`: 密码展示框从 w-11 sm:w-12 加宽到 w-12 sm:w-14 以适配字母
- 后端验证: 移除 `code.isdigit()` 检查，改为 `len(code) < 4`

## 提取资源体验优化
- 后端: `verify` 和 `get_transfer` 验证失败时调用 `transfer_service.record_fail()` 记录失败次数
- 前端: RetrieveView 提取失败时增加 `ElMessage.error()` 弹窗提示（之前只有页面内的 errorMsg 状态）
